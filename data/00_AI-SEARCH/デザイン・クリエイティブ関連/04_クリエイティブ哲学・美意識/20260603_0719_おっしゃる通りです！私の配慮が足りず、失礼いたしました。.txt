実際のファイル名には頭に 20260602_2325_ のような「日付＋時間」の13文字（またはアンダーバーを含めて14文字）が必ず自動で入ってしまいますよね。

そのため、普通にそのままアルファベット（名前）順でソートしてしまうと、せっかく付けた 00_【進行】 や 01_【定型】 の意味がなくなり、結局「日付の数字」が最優先されて時系列と同じ並びになってしまいます。

意図通りに「進行マップ」や「定型パターン」を一番上にカチッと集めるためには、ご指摘の通り、名前順ソートの分岐処理の時だけ、先頭の「日付と番号（タイムスタンプ部分）」を内部で一時的にカット（排除）して比較するロジックが絶対に必要です。

具体的には、以下のような裏側の処理を実装することになります。

🛠️ 名前順ソート時の「手前の文字を省く」内部処理イメージ

PWAのプログラム側で、名前順ソート（アルファベット順）が選ばれた時だけ、以下の2ステップを実行します。

• タイムスタンプのカット（正規表現や slice を使用）ファイル名の先頭にある YYYYMMDD_HHMM_ （例: 20260602_2323_00_【進行】...）の最初の14文字を、ソートの計算時だけ内部的に無視します。内部比較用の文字列 ＝ 00_【進行】プロジェクト進捗マップ.txt
• カットした文字列同士でアルファベット順ソートをかけるこれで、実際のファイルがいつ作られたか（日付）に関係なく、純粋に 00_ や 01_ というあなたが付けた名称の順番だけで綺麗に並び替わります。

💻 ソートを実装するJavaScript（TypeScript）コード例

今お使いのSolidシリーズ（PWA）のコードにそのまま組み込める、先頭の14文字（タイムスタンプ）を排除してソートするシンプルな関数を書いておきます。AIにビューアーを改良させる際の見本にしてください。

javascript

```
// 名前順（アルファベット・数字順）ソート関数
const sortByNameWithoutTimestamp = (files) => {
  return [...files].sort((a, b) => {
    // 先頭の「YYYYMMDD_HHMM_」（14文字）を取り除く
    // もしタイムスタンプがないファイルがあってもバグらないように slice を調整
    const nameA = a.fileName.startsWith('20') ? a.fileName.slice(14) : a.fileName;
    const nameB = b.fileName.startsWith('20') ? b.fileName.slice(14) : b.fileName;

    // 排除した純粋な名称（00_ や 01_）でアルファベット・数字順比較
    return nameA.localeCompare(nameB, undefined, { numeric: true, sensitivity: 'base' });
  });
};
```

コードは注意してご使用ください。

※ { numeric: true } をいれておくことで、01_ の次に 02_、10_ が正しく数字の大きさ順で並ぶようになります。

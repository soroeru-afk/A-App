Vite(React)アプリのGitHub Pages公開 スムーズ化ガイド

今回のように、「新しいアプリを作ってGitHubへ公開し、GitHub Pages (Visit site) で見れるようにする」という作業を今後もスムーズに行うための標準ワークフローをまとめました。

次回以降、別のツールやアプリを作るときは、この手順に沿って進めることで「画面が真っ白になる」問題や「コマンドが見つからない」問題を防ぐことができます。

1. アプリのディレクトリ内に起動用バッチファイルを配置する

今回 SUPER-FOLDER-LOG-VIEWER- フォルダ内に配置したように、今後はアプリを新規作成するたびに、そのアプリのフォルダ内に以下の2つの .bat ファイルを置いておくことをお勧めします。

1_開発スタート.bat
2_ビルドとプレビュー.bat

こうすることで、他のプロジェクトと混ざらず、対象のフォルダ内でダブルクリックするだけで確実に起動できます。

2. GitHub Pages 公開用の Vite 設定（ここが真っ白になる原因！）

GitHub Pages は、URLが https://ユーザー名.github.io/リポジトリ名/ となるため、パスの基準がずれてしまいます。これを防ぐために、毎回必ず vite.config.ts に base を追加してください。

typescript
// vite.config.ts の例
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
export default defineConfig({
  // 👇 これを必ず追加する！(相対パス指定になるため、真っ白を防ぎます)
  base: './', 
  
  plugins: [react()],
});
3. 今後のワークフロー（AIへの指示の出し方）

次回新しいアプリを作る際、AIに以下のように依頼すると一発で全ての設定が整います。

「新しいReactアプリを作って。GitHub Pagesで公開する予定なので vite.config.ts の base: './' 設定もお願い。あと、フォルダの中にダブルクリックで起動できる 1_開発スタート.bat も作っておいて。」

この一言を添えるだけで、今回苦労した「設定漏れ」や「起動の手間」を最初からショートカットできます！ぜひ今後のプロンプト（指示）のテンプレートとしてお使いください。

このガイドにも書いてありますが、次回以降は私（AI）に対して、

「新しいReactアプリを作って。GitHub Pagesで公開するから vite.config.ts に base: './' を入れておいて。あと、フォルダの中にダブルクリックで起動できるバッチファイルもよろしく」

という風に指示を出していただければ、面倒な設定やショートカットの配置を一発で終わらせるように動きます。 これで毎回「画面が真っ白だ」「コマンドが打てない」といったタイムロスをなくし、かなりスムーズに作業が進められるはずです！ぜひご活用ください。

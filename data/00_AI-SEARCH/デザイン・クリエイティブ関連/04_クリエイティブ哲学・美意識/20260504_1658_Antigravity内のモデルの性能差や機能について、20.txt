Antigravity内のモデルの性能差や機能について、2026年現在の最新情報をまとめました。

Gemini 3 シリーズの性能差

Antigravity で利用できるモデルは、目的や「思考の深さ」によって使い分けられます。

モデル性能・特徴主な用途

Gemini 3.1 Pro (High)最高峰の推論力。複雑なロジックの構築や、大規模なコードベース全体の設計に最適。ARC-AGI-2 ベンチマークで高いスコア（77.1%）を記録。難解なデバッグ、新規機能の設計、大規模リファクタリング。

Gemini 3.1 Pro (Low)バランス重視。High ほどの思考時間はかけないが、十分な精度を保ちつつレスポンスが速い。日常的なコーディング、中規模の修正、技術的な相談。

Gemini 3 Flash圧倒的なスピード。推論の深さは Pro に譲るが、単純な作業やクイックな修正では Pro を上回る。軽微なバグ修正、コードの要約、ボイラープレートの生成。

Antigravity の機能と可能性

GitHub 同期や PWA 対応についても、このツールの特徴から回答します。

• GitHub 同期とデザイン変更:Antigravity はエージェント型 IDE として設計されており、複数ファイルの同時編集やターミナル操作が可能。GitHub との同期や、UI/UX の複雑なデザイン変更（CSS/フレームワークの調整）は得意分野の一つ。
• PWA などの複雑な開発:「自律型エージェント」として、計画からコーディング、ブラウザでの動作確認まで実行できるため、PWA（Progressive Web Apps）のようなマニフェストファイルやサービスワーカーの構築を含む開発も可能。特に Gemini 3.1 Pro (High) を使用すれば、複雑な構成の提案も期待できる。
• 上位モデルが制限された場合の代替:Pro モデルには利用制限（クォータ）がある。制限に達した際に Gemini 3 Flash や Claude 4.6 シリーズに切り替えて作業を続けるのは、多くのユーザーが推奨する活用法。Flash でも、指示が明確なタスクであれば PWA のパーツ作成などは問題なくできる。

Qiita の活用ガイド などのコミュニティ記事では、普段は Flash で使用し、行き詰まった時だけ Pro (High) を使用することが推奨されている。

# 20260611_2159_- SOLID SQUARE CALENDAR_音声読み上げバグ修正および動的ボイス移行完了

`SOLID SQUARE CALENDAR` （カレンダーログビューアー）において、音声読み上げ（SpeechSynthesis）の初期ロード不具合の修正、および日本語ボイスの動的取得への移行が完了しました。

## 実施した作業概要

- **音声ロード遅延バグの修正 (`App.tsx` & `EditorModal.tsx`)**:
  - ページ読み込み直後に音声リストのロードが遅れる Web Speech API のブラウザ仕様に対応するため、アプリ起動時（`App.tsx` のマウント時）および編集画面表示時（`EditorModal.tsx` のマウント時）に、事前に `window.speechSynthesis.getVoices()` をダミー呼び出し（空撃ち）してブラウザにプリロードを促す処理を実装しました。
  - `speechSynthesis.onvoiceschanged` イベントをハンドリングし、読み込み完了と同時にデータをキャッシュさせるようにしました。
- **実在する日本語音声のみの動的リスト化 (`App.tsx`)**:
  - 従来設定されていた「Keita」「Nanami」などが特定の環境で全くまともな声にならなかった問題を解決するため、静的な固定リストを廃止しました。
  - ブラウザが実際にサポートしている日本語音声（`ja-JP` / `ja_JP`）のみを起動時に検出・抽出し、設定メニュー（セレクトボックス）の選択肢に動的にリストアップする「動的ボイス読み込み」へ移行しました。
  - セレクトボックス上の表記から `Microsoft` や `Google`、`Online` などの余計な文字列を自動で切り取り、すっきりとした大文字のソリッドなデザインで統一しました。
  - もし設定された音声名が存在しない環境（Chrome等）で開いた場合は、自動的に利用可能な最初の日本語音声に代替選択されるインテリジェントな構成としました。
- **カレンダー配置レイアウトの微調整 (`GridSqView.tsx`)**:
  - 前ステップでアスペクト比固定時にカレンダー全体が画面中央に配置されることで、ヘッダー行（曜日）の位置が下に離れてしまう違和感を解消するため、縦の整列を「上寄せ（`items-start`）」に変更しました。
  - これにより、曜日ヘッダーは常にタイトル「04 CALENDAR BANKS」のすぐ下の定位置に固定され、カレンダー本体のみが下部へ綺麗な比率で伸びる自然なレイアウトになりました。

## 動作確認・デプロイ結果

- `npm.cmd run build` を実行し、TypeScriptのビルドが成功することを確認しました。
- 変更内容をコミットし、GitHubへプッシュしました。GitHub Actionsによるデプロイが正常に成功し、Web上で動作することを確認済みです。

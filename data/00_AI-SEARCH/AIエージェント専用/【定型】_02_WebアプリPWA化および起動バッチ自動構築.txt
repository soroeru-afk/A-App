# 【定型】_02_WebアプリPWA化および起動バッチ自動構築

このファイルは、開発したWebアプリやツールをPWA（Progressive Web App）化し、ダブルクリックで起動・プレビューできるバッチファイルを自動生成するための定型処理手順書です。
ユーザーから「PWA化と起動バッチ構築のパターンで処理して」と指示があり、対象のプロジェクトフォルダが指定された場合に適用します。

---

## 1. 処理手順 (Workflow Steps)

### ステップ 1: プロジェクト環境の確認
- 対象プロジェクトの構成（React + ViteなどのNode.js環境か、純粋なHTML/JSの静的構成か）を解析します。
- すでにPWA化されている場合は、現在のService Workerやマニフェストの状態を確認します。

### ステップ 2: PWA用アセットの自動生成・設定
1. **マニフェストファイル (`manifest.json` または `site.webmanifest`) の作成**:
   - アプリ名、テーマカラー（ソリッドな暗色系 `#0E1014` 推奨）、スタートURL、表示モード（`standalone`）等を定義します。
2. **アイコンの自動最適化生成**:
   - ユーザーから提供された、あるいは既存のロゴ画像から、PWA必須サイズ（`192x192` および `512x512`）のPNGアイコンを自動リサイズ・透過/黒枠処理して生成します。
3. **サービスワーカー (`sw.js`) の実装**:
   - オフライン対応およびキャッシュ問題（しぶとい古いキャッシュの打破）に対応するため、キャッシュ制御戦略（Network-First等）を組み込んだサービスワーカーを自動実装します。
   - `sw.js` のキャッシュバージョン（例: `v1.0`）を適切に管理します。

### ステップ 3: 起動・ビルド用バッチファイルの自動生成
プロジェクトのルートフォルダ直下に、ダブルクリックで動作する以下のWindowsバッチファイルを自動作成します（相対パス `%~dp0` を用いた堅牢な記述）。

1. **`開発スタート.bat` (または `start_dev.bat`)**:
   - ローカル開発サーバーを起動するバッチ（例: `npm run dev` または Pythonの簡易HTTPサーバー起動）。
2. **`ビルド.bat` (または `build.bat`)**:
   - 本番用アセットをビルド・コンパイルするバッチ（例: `npm run build`）。
3. **`プレビュー.bat` (または `preview.bat`)**:
   - ビルドされた成果物をローカルでプレビュー確認するためのサーバー起動バッチ（例: `npm run preview`）。

### ステップ 4: 自動検証と起動テスト
- 実際に起動バッチやビルドコマンドを実行し、エラーなくビルドが通るか、ローカルサーバーが正常に起動してPWA機能が動作しているかを自動検証します。

### ステップ 5: 進捗マップと実行ログの更新
- 完了後、以下の2つの出力を実行します。
  1. `00_【進行】 プロジェクト進捗マップ.txt` を読み込み、最新の進捗状況を上書きまたは下部に追記します。
  2. `YYYYMMDD_HHMM_- PWA化および起動バッチ構築完了.txt` という名前で、今回の実装完了のログを新規出力します。
  - ログには、作成したバッチファイル名やPWAのバージョン情報を、音声読み上げしやすい綺麗なMarkdown形式で記載してください。

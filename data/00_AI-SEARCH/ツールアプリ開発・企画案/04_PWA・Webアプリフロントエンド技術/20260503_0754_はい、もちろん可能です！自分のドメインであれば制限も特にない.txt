はい、もちろん可能です！自分のドメインであれば制限も特にないので、自由にツールを自作して動かせますよ。

具体的には、以下の手順で進めるのがスムーズです。

1. 動作対象を自分のドメインに指定する

Tampermonkeyスクリプトの冒頭（メタデータブロック）にある // @match や // @include に、自分のサイトのURLを記述します。

javascript

```
// ==UserScript==
// @name         マイサイト用便利ツール
// @namespace    http://tampermonkey.net
// @version      0.1
// @description  自分のサイトを便利にする
// @author       You
// @match        https://your-domain.com*  <-- ここに自分のドメインを書く
// @grant        none
// ==/UserScript==

(function() {
    'use strict';
    // ここに実行したいコード（ボタン追加や表示変更など）を書く
    console.log("マイツールが起動しました！");
})();
```

コードは注意してご使用ください。

2. メリットと使い道

自分のサイトに対してTampermonkeyを使うと、以下のようなときに便利です。

• 管理画面の効率化: サイトの管理画面（CMSなど）に、標準機能にはない「一括削除ボタン」や「文字数カウント」を無理やり追加する。
• 開発中のデバッグ: 本番環境のソースはいじらずに、フロントエンドの修正案を自分のブラウザ上だけで試してみる。
• 非公開のメモ機能: サイト上の特定の場所に、自分にしか見えないメモ欄を表示させる。

注意点：セキュリティ（CORS）

もしスクリプト内で「別のドメイン」からデータを取ってきたりする場合は、Tampermonkeyの // @grant GM_xmlhttpRequest を使う必要があります。これを使えば、通常のJavaScriptでは制限される「クロスドメイン通信」も突破できるので、外部APIとの連携もしやすくなります。

★ SUPER FOLDER LOG VIEWER のファイル構造

カオルさま、ご自身のツールを見せていただきありがとうございます。 非常に多機能でモダンな「ローカルフォルダ内のテキスト・ログ管理ビューア」ですね。

コードを拝見したところ、**HTML・CSS・JavaScriptが1つのファイルにまとめられた、独立したシングルページアプリケーション（SPA）**として非常に綺麗に設計されています。

プログラムの全体的な構造と、採用されている技術・仕組みを整理してみました。

1. 全体構成（1ファイル完結型）

外部のライブラリやフレームワーク（ReactやVueなど）に依存せず、ブラウザの標準機能（Vanilla
JS）だけで作られています。これにより、オフライン環境やローカルにポンと置くだけですぐに動く、非常に軽量でポータブルなツールになっています。

2. データと権限の管理（JavaScript）

モダンなブラウザAPIを駆使して、ローカルファイルの読み書きと設定の保存を行っています。

  - File System Access API (showDirectoryPicker)
    ブラウザから直接ローカルのフォルダを指定し、中のファイルを読み込んだり、新規作成・編集・削除・移動を行うための核となるAPIです。
  - IndexedDB (LogViewerDB)
    一度開いたフォルダの「ハンドル（アクセス権）」を保存しています。これにより、次回ツールを開いた際に「前回のフォルダーを開く」ボタンからスムーズに再接続できるようになっています。
  - LocalStorage (lv_theme など) 文字サイズ、フォント、テーマ、サイドバーの幅などの「UI設定」をブラウザに記憶させています。

3. 画面レイアウト・UI（HTML/CSS）

画面は大きく「サイドバー」と「メインコンテンツ」に分かれ、各種パネルが重なるように配置されています。CSS変数（:root）を巧みに使って、テーマの切り替えを瞬時に行える構造になっています。

  - サイドバー (#sidebar)
    フォルダ内のファイル一覧を表示。ファイルはただ並べるだけでなく、フォルダ名（カテゴリ）や、ファイル名・更新日時から「月」「日」を自動判定してグループ化（アコーディオンUI）する高度なロジックが組まれています。
  - メインコンテンツ (#main)
    選択したファイルの内容を表示します。通常のテキスト表示だけでなく、「名前：セリフ」の形式を自動で検知してチャットのような吹き出しUIに変換するユニークな機能が備わっています。
  - 各種フローティングパネル
    設定パネル（#settings-panel）、一括操作バー（#bulk-bar）、移動先選択パネルなど。これらは絶対位置指定やリサイズ計算関数（positionPanel）で、ボタンの近くに適切にポップアップするように制御されています。

4. 主要な機能（JavaScriptのロジック）

状態変数（allFiles, currentFileObj, selectedFiles など）を中心に、以下のような操作を実現しています。

1.  読み込み・パース (loadFiles, parseFilename) ファイル名から日付やタイトルを抽出し、見やすく整理。
2.  コンテンツのレンダリング (renderContent, extractFirstSentence)
    チャット風のUI変換に加え、本文の1行目を自動で「大見出し」として抽出する機能があります。
3.  ファイル管理 (moveToRoot, bulkMoveTo, deleteCurrentFile, renameCurrentFile など)
    単一ファイルだけでなく、チェックボックス風の選択モードを用いた「一括移動」「一括削除」まで網羅しています。
4.  リアルタイムプレビュー (applyFontSize, applyTheme など)
    設定パネルのスライダーを動かすと、リアルタイムで画面のフォントやレイアウト（CSS変数）が書き換わる仕組みです。

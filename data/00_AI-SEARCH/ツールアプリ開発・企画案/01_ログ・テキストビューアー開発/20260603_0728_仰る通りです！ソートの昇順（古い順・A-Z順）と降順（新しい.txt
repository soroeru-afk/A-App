仰る通りです！ソートの昇順（古い順・A-Z順）と降順（新しい順・Z-A順）をひっくり返す「逆並び（トグル）機能」は絶対にあった方がいいですね。

特に日付ソートの場合、「最新のログを一番上で見たい時（降順）」と「過去の履歴を上から物語のように音声読み上げさせたい時（昇順）」で上下を逆にする必要が必ず出てきます。

ボタンを何個も作るとUIがごちゃつくので、ご提案の通り「日付順」「名前順」の2つのボタンを用意し、同じボタンをもう一度押すごとに矢印（▲▼）と並び順が反転する仕組みにするのが、SolidシリーズのミニマルなUIにも一番馴染みます。

🔄 逆並び（トグル）を含めたUIと挙動のイメージ

ボタンの表示が以下のようにパタパタ切り替わるイメージです。

• [ 🕒 日付順 ▼ ] （デフォルト：最新が一番上）もう一度押すと ➔ [ 🕒 日付順 ▲ ] （過去が一番上：一気読み・連続読み上げに便利）
• [ 🔤 名前順 ▲ ] （00_進行 や 01_定型 が一番上）もう一度押すと ➔ [ 🔤 名前順 ▼ ] （通常の実行ログが上にきて、定型が下に隠れる）

💻 逆並びに対応したPWA（JavaScript）のロジック例

裏側の処理では、現在「昇順（1）」か「降順（-1）」かの状態（ステート）を1つ持たせておき、最後にソート結果にその乗数をかけるだけで、手前のタイムスタンプを省く処理を維持したまま、綺麗に逆並びが実現できます。

javascript

```
// ソート状態の管理イメージ
let isAscending = true; // trueなら昇順(▲)、falseなら降順(▼)

const sortFiles = (files, mode) => {
  const direction = isAscending ? 1 : -1;

  return [...files].sort((a, b) => {
    if (mode === 'date') {
      // 通常の日付ソート（逆並び対応）
      return a.fileName.localeCompare(b.fileName) * direction;
    }

    if (mode === 'name') {
      // 先頭14文字（タイムスタンプ）を排除した名前ソート（逆並び対応）
      const nameA = a.fileName.startsWith('20') ? a.fileName.slice(14) : a.fileName;
      const nameB = b.fileName.startsWith('20') ? b.fileName.slice(14) : b.fileName;
      return nameA.localeCompare(nameB, undefined, { numeric: true }) * direction;
    }
  });
};
```

コードは注意してご使用ください。

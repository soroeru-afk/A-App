# AIエージェント自律ファイル仕分け・同期プロトコル (AGENTS.md)

このドキュメントは、AIエージェント（Antigravity）が別端末や異なるセッションで開発を再開・継続する際に、進捗状況と企画内容を完全に同期し、自律的にログを分類保存するためのルールです。

エージェントはセッション開始時に必ず本ドキュメントを読み込み、以下の手順に従ってください。

---

## 1. 起動・作業開始時の初期シークエンス（必須）

何か作業を始める前に、必ず以下の手順を実行してください。

1. **Google ドライブ of ルートパスの確認とファイルロード**:
   ローカルのGoogle ドライブにある、以下のルートフォルダをスキャンしてください。
   `G:\マイドライブ\00_AI-SEARCH\AIエージェント専用\`
   このフォルダは、**すべてのファイルを1階層フラットに配置**する運用とします。サブフォルダは作成しないでください。
   
2. **進捗マップの読み込みと同期**:
   フォルダ直下にある以下のファイルを読み込み、現在の開発フェーズ、最新 of 最新の進捗、次に実行すべきタスクを完全に同期してください。
   `00_【進行】_プロジェクト進捗マップ.txt`

3. **Gitリポジトリの最新同期（超重要）**:
   別環境での作業漏れやデグレードを防ぐため、作業開始前に必ず対象のGitリポジトリで **`git pull origin main` を実行**し、リモートの最新コードをローカル環境へ同期・反映してください。

4. **GitHubリポジトリとローカル/Drive情報の不一致確認と報告**:
   - 進捗マップや完了ログに記載されている「最新機能」と、GitHub（リモート/ローカル）の最新コードに乖離（自宅と事務所間での未プッシュ等）がないか、必ず開始時に確認してください。
   - もし、進捗マップに記述がある変更内容がGitHubの最新コードに存在しない場合、**推測で無理にその機能を再作成・構築しようとせず、速やかに作業を一時停止してください。**
   - その上で、ユーザーに「進捗マップに記載されている最新機能がGitHubにプッシュ（更新）されていないため、未反映です。」と明確に報告し、プッシュの有無や指示を確認してください。

## 2. 進行中プロジェクトの進捗管理ルール (上書き・追記運用)

現在進行中のプロジェクトの全体スケジュールや最新の進捗は、フォルダの直下に以下のファイル名で管理します。
- **ファイル名**: `00_【進行】_プロジェクト進捗マップ.txt`
  - 増えた場合は `00-01_【進行】_〜.txt`、`00-02_【進行】_〜.txt` のように枝番を付与し、アンダーバーで区切ります。
- **タイトル（1行目）**: `# 00_【進行】_プロジェクト進捗マップ` のように、ファイル名と同様に `00_【進行】_` で始めて記述してください。
- **運用方法**: 新しいタスクの追加やステータス変更があるたびに、既存 of ファイルの中身を**「上書き、または下部へ追記」して1ファイルで維持**してください。

## 3. 進捗記録と「新規ログファイル」出力に関する厳格なルール

**【重要】** 『UIの微調整』や『小さな機能追加』『単発のトラブルシューティング』などの細かい作業（フェーズの途中の作業）の段階では、**絶対に新規ログファイルを作らない**でください。

### A. 細かい作業経過の記録（進捗マップへの直接追記）
- 日常的な作業、細かい修正やトラブルシューティングの経過と結果は、すべて `00_【進行】_プロジェクト進捗マップ.txt` 内の該当プロジェクトのエリア（下の行など）に、**日付と進捗ステータスとして直接追記・更新**していってください。
- これらの中間作業では、新規のテキストファイルは出力しません。

### B. 新規ログファイル【構築完了ログ】を出力する条件
- 新規のログファイルを作っていいのは、**システム全体が完成した時や、大きなマイルストーンが『完了』した時のみ**とします。
- この場合のみ、詳細な「実行経過・構築完了ログ」を新規ファイルとして書き出してください。
- **ファイル名規則**: `YYYYMMDD_HHMM_- プロジェクト名_タスク名.txt` (日付プレフィックス + `- ` + プロジェクト名 + `_` + 内容)
  - *例*: `20260604_2331_- VOICEVOX Studio_最新データのGitHub同期完了.txt`
  - *例*: `20260604_1625_- SUPER FOLDER LOG VIEWER_縦書き機能およびレイアウト調整.txt`
- **タイトル（1行目）**: ファイル名と同様の形式で記述してください（マークダウンの見出し `#` で開始）。
  - *例*: `# 20260604_2331_- VOICEVOX Studio_最新データのGitHub同期完了`
  - *例*: `# 20260604_1625_- SUPER FOLDER LOG VIEWER_縦書き機能およびレイアウト調整`
- **保存先**: `G:\マイドライブ\00_AI-SEARCH\AIエージェント専用\` の直下
- **ログの記載内容**:
  - 完了したシステム/マイルストーンの概要と成果
  - 変更したファイルやコードの要点
  - 発生した課題とその解決策
  - 次のフェーズへの申し送り事項
- **フォーマット**: 見出し（`#`）や太字（`**`）を用いたMarkdown形式で、人間がビューアーで音声読み上げさせても一発で理解できる綺麗なサマリーにまとめてください。

## 4. 定型パターン処理機能ルール

ユーザーから定型化されたワークフローを実行する指示があった場合の処理手順です。

1. **【定型】ファイルの検知**:
   ユーザーが `G:\マイドライブ\00_AI-SEARCH\AIエージェント専用\` 直下に `【定型】_01_タスク名.txt` などのファイルを配置することがあります。
   - 連番「_01_」「_02_」「_03_」を付与して量産・作成され、アンダーバーで区切ります。
   - *例*: `【定型】_01_ライブドアブログ自動投稿.txt`
   - **タイトル（1行目）**: `# 【定型】_01_ライブドアブログ自動投稿` のように連番と【定型】を含め、アンダーバーで区切ってください。
   
2. **トリガーと読み込み**:
   チャットで「〇〇のパターンで処理して」と指示が出たら、エージェント（マルチエージェント）はまず該当する【定型】ファイルを最優先で読み込んでください。
   
3. **自律的タスク割り振りと実行**:
   そのファイルに記載されている処理ステップに従って、各サブエージェントにタスクを自律的に割り振って実行してください。
   
4. **進捗・結果 of 出力**:
   実行中および完了時の進捗は、厳格化されたルールに従い、細かい作業はすべて『00_【進行】_プロジェクト進捗マップ.txt』へ直接上書き・追記してください。大きなマイルストーンやシステム全体が完成した際のみ、新規の構築完了ログファイル（日付プレフィックス + `- ` + プロジェクト名 + `_` + 内容）を作成して出力してください。
